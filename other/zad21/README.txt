n - броя на редовете и колоните
br - броя на диагоналите, успоредни на главния (без него)
dolni - флаг за ненулеви елементи за диагонали под главния
gorni - флаг за ненулеви елементи за диагонали над главния
count - броя диагонали с нулеви елементи, успоредни на главния

намиране на броя br(намиране на зависимостта):
-> при n=2 => 2 диагонала
-> при n=3 => 4 диагонала
-> при n=4 => 6 диагонала
-> при n=5 => 8 диагонала
=> br=n+(n-2)

пример:
2 5 0 2 3
0 3 6 0 6
2 0 4 7 0
5 6 0 5 8
0 2 1 0 6

диагонали успоредни на главния:
   стойности  =>  индекси
1. 0, 0, 0, 0 => [1][0], [2][1], [3][2], [4][3]
2. 5, 6, 7, 8 => [0][1], [1][2], [2][3], [3][4]
3. 2, 6, 1 => [2][0], [3][1], [4][2]
4. 0, 0, 0 => [0][2], [1][3], [2][4]
5. 5, 2 => [3][0], [4][1]
6. 2, 6 => [0][3], [1][4]
7. 0 => [4][0]
8. 3 => [0][4]

n=5; br=8; br/2=4

зависимост: елементите на диагоналите под главния, имат огледални индекси на тези над главния
затова ги обхождаме едновременно:
итерация на цикъла:

k=1; k<=br/2:
	i=k, j=0; i<n, j<n-k:
		k=1: (0, 0, 0, 0 & 5, 6, 7, 8)
			i=1; j=0: [1][0] - 0, [0][1] - 5
			i=2; j=1: [2][1] - 0, [1][2] - 6
			i=3; j=2: [3][2] - 0, [2][3] - 7
			i=4; j=3: [4][3] - 0, [3][4] - 8

		k=2: (2, 6, 1 & 0, 0, 0)
			i=2; j=0: [2][0] - 2, [0][2] - 0
			i=3; j=1: [3][1] - 6, [1][3] - 0
			i=4; j=2: [4][2] - 1, [2][4] - 0

		k=3: (5, 2 & 2, 6)
			i=3; j=0: [3][0] - 5, [0][3] - 2
			i=4; j=1: [4][1] - 2, [1][4] - 6

		k=4: (0 & 3)
			i=4; j=0: [4][0] - 0, [0][4] - 3

след това ако в даден диагонал срещне ненулев елемент, флага става равен на 1, и след като приключи проверката на текущия диагонал, count не се увеличава!
Ако пък при проверка на диагонала не е срещнат ненулев елемент, флага си остава равен на 0, и след края на обхождането на текущия диагонал count се увеличава с 1!
